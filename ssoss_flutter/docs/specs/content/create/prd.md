## 1. 배경 및 목적

**배경**

소상공인·가게 운영자가 SNS/블로그에 올릴 콘텐츠를 직접 작성하는 데 시간이 많이 든다. 업로드 채널·목적·톤·강조 내용을 단계적으로 입력하면 AI가 맞춤 콘텐츠를 생성해 주는 흐름이 필요하다.

**목적**

사용자가 3단계 입력 플로우로 생성 조건을 지정하고, 서버 API를 통해 AI 콘텐츠를 생성할 수 있게 한다. 선택 항목(채널·목적·톤)은 enum으로 정의해 이후 API payload 매핑을 유연하게 변경할 수 있도록 한다.

---

## 2. 사용자 스토리

- [ ] As a **가게 운영자**, I want to **업로드 채널·목적·작성 톤을 선택**, so that **채널과 목적에 맞는 콘텐츠를 받을 수 있다**.
- [ ] As a **가게 운영자**, I want to **강조 내용·금지 내용·키워드를 입력**, so that **원하는 메시지와 제약을 반영한 콘텐츠를 생성할 수 있다**.
- [ ] As a **가게 운영자**, I want to **콘텐츠 생성하기를 눌러 서버에 생성을 요청**, so that **입력한 조건으로 AI 콘텐츠를 받을 수 있다**.
- [ ] As a **가게 운영자**, I want to **생성 중 화면에서 나가기로 홈에 복귀**, so that **생성을 기다리지 않고 앱을 계속 사용할 수 있다**.

## 3. 요구사항

### 3.1 기능 요구사항 (Functional Requirements)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01 | 콘텐츠 생성은 **3단계 입력 + 생성 중** 플로우로 구성한다. (01 기본 설정 → 02 콘텐츠 설정 → 03 세부 설정 → 생성 중) | Must |
| FR-02 | 상단 진행 바로 현재 단계를 표시한다. (`01 기본 설정` / `02 콘텐츠 설정` / `03 세부 설정`) | Must |
| FR-03 | **1단계(기본 설정)**: 업로드 채널을 단일 선택한다. 선택이 완료되어야 `다음` 버튼이 활성화된다. | Must |
| FR-04 | **2단계(콘텐츠 설정)**: 업로드 목적·콘텐츠 작성 톤을 각각 단일 선택한다. **두 항목 모두** 선택되어야 `다음` 버튼이 활성화된다. | Must |
| FR-05 | **3단계(세부 설정)**: 강조 내용(필수), 금지 내용(선택), 주요 키워드(선택), 사진 첨부 가이드(선택)를 입력한다. | Must |
| FR-06 | 3단계에서 **강조 내용**이 비어 있지 않을 때만 `콘텐츠 생성하기` 버튼이 활성화된다. (금지 내용·키워드·사진 가이드는 활성화 조건에 포함하지 않는다) | Must |
| FR-07 | 비활성 하단 버튼은 Primary 연한 색, 활성 시 Primary 본색으로 표시한다. | Must |
| FR-08 | 주요 키워드는 입력 필드에 한 개씩 입력 후 `추가하기`를 누르면 `#`가 붙은 태그(칩)로 추가된다. | Must |
| FR-09 | 키워드 태그의 `X`를 누르면 해당 키워드를 목록에서 제거한다. | Must |
| FR-10 | 빈 문자열·공백만 있는 키워드는 추가하지 않는다. (이미 `#`만 있는 경우 등) | Should |
| FR-10a | 주요 키워드(해시태그)는 **최대 10개**까지 추가할 수 있다. 한도 초과 상태에서 `추가하기`를 누르면 토스트로 안내하고, 입력값은 유지한다. (입력·버튼은 비활성화하지 않는다) | Must |
| FR-11 | `사진 첨부 가이드`는 체크박스로 on/off 토글한다. (선택) | Must |
| FR-12 | `콘텐츠 생성하기` 탭 시 1~3단계에서 수집한 값을 바탕으로 콘텐츠 생성 API를 호출하고 **생성 중 화면**으로 이동한다. | Must |
| FR-13 | 업로드 채널 · 업로드 목적 · 콘텐츠 작성 톤은 **domain enum**으로 정의하고, API 전송 시 enum → 서버 기대 값으로 변환 가능한 구조를 둔다. | Must |
| FR-14 | 생성 중 화면: 앱바 뒤로가기(back)는 제거하고, 우측 `나가기`만 제공한다. | Must |
| FR-15 | 생성 중 화면에서 `나가기`를 누르면 홈 화면으로 이동한다. | Must |
| FR-16 | 생성 중 화면의 로딩 이미지는 추후 GIF 교체를 위해 **플레이스홀더 컨테이너**로 둔다. (정적/빈 컨테이너 허용) | Must |
| FR-17 | 생성 중 화면에는 하단 `콘텐츠 생성하기` 버튼을 **배치하지 않는다**. | Must |
| FR-18 | 생성 중 화면 앱바에는 타이틀을 두지 않는다. 본문에 `콘텐츠 발행 중이에요`와 보조 문구(크레딧 차감 안내)를 표시한다. | Must |
| FR-19 | 1~3단계 앱바는 뒤로가기 + 타이틀 `콘텐츠 생성`을 제공한다. 뒤로가기는 이전 단계(또는 진입 전 화면)로 이동한다. | Must |
| FR-20 | 생성 완료 후 결과 화면 전환은 본 PRD 범위에서 명세하되, 결과 UI 상세는 Out of Scope로 둘 수 있다. (생성 요청·생성 중 UI까지 Must) | Should |

> **우선순위 기준** — Must: 반드시 / Should: 가급적 / Could: 여유 시

#### Enum 정의 (UI 라벨 → enum · API 값 매핑 분리)

| Enum | UI 옵션 | 비고 |
|------|---------|------|
| `UploadChannel` | 블로그 / 스레드 / 당근 / 인스타그램 | 1단계. 와이어프레임 `업로드 SNS 채널 선택` 기준. hi-fi 확정 시 옵션 조정 |
| `UploadPurpose` | 정보성 / 이벤트·할인 / 신메뉴·홍보 | 2단계 |
| `ContentTone` | 일상형 / 감성형 / 정보형 / 홍보형 | 2단계. UI에 보조 설명 문구 표시 |

> enum 식별자(code)와 서버 payload 문자열은 분리한다. UI 라벨 변경·API 스펙 변경 시 mapper만 수정하면 된다.

#### 3단계 필드 상세

| 필드 | 필수 | UI |
|------|------|-----|
| 강조 내용 | 필수 | multiline 입력. placeholder: `콘텐츠에 들어갈 내용을 작성해주세요.` |
| 금지 내용 | 선택 | multiline 입력. 라벨 옆 `선택`. placeholder: `콘텐츠에 들어가면 안되는 내용을 작성해주세요.` |
| 주요 키워드 | 선택 | 입력 + `추가하기` → `#키워드` 칩. 안내: `가게 정보와 저장된 키워드를 참고해 콘텐츠를 생성해요` |
| 사진 첨부 가이드 | 선택 | 체크박스. 안내: `해당 글에 어울리는 사진 가이드를 드려요.` |

## 4. 범위 (Scope)

### In Scope (이번 구현에 포함)

- 3단계 입력 위저드 UI 및 진행 바
- 단계별 CTA 활성/비활성 규칙
- 업로드 채널 · 목적 · 톤 enum 및 API 매핑 구조
- 키워드 추가/삭제(#칩)
- 콘텐츠 생성 API 호출 및 생성 중 화면
- 생성 중: 뒤로가기 제거, `나가기` → 홈, 로딩 플레이스홀더, 하단 버튼 미포함

### Out of Scope (이번 구현에서 제외)

- 생성 완료 결과 화면·복사하기 등 후속 UX (이유: 별도 스펙으로 분리 예정)
- 생성 중 로딩 GIF 에셋 반영 (이유: 추후 교체. 이번엔 컨테이너만)
- 가게 정보/저장된 키워드 자동 불러오기 UI (이유: 안내 카피만 포함, 연동은 후속)
- 크레딧 차감 상세 UI·잔액 표시 (이유: 안내 문구만 표시)
- 콘텐츠 수정·재생성·히스토리 (이유: 별도 기능)

**주요 사용자 흐름**

```
[홈/진입] → [1단계 기본 설정: 업로드 채널 선택]
  → (채널 미선택: 다음 비활성)
  → (채널 선택: 다음 활성) → 다음

→ [2단계 콘텐츠 설정: 목적 + 톤 선택]
  → (둘 중 하나라도 미선택: 다음 비활성)
  → (둘 다 선택: 다음 활성) → 다음

→ [3단계 세부 설정]
  → 강조 내용 입력(필수) / 금지·키워드·사진 가이드(선택)
  → 키워드: 입력 → 추가하기 → #칩 표시 / X로 삭제
  → (강조 내용 비어 있음: 콘텐츠 생성하기 비활성)
  → (강조 내용 입력됨: 콘텐츠 생성하기 활성) → 탭

→ 생성 API 호출 + [생성 중 화면]
  → 나가기 → [홈]
  → (생성 성공) → [결과 화면 — Out of Scope / 후속]
  → (생성 실패) → 오류 안내 후 재시도 또는 이전 단계 복귀
```

**생성 중 화면 UX 규칙**

- 앱바: 뒤로가기·타이틀 없음 + 우측 `나가기` (Primary 계열 텍스트). 하단 CTA 없음
- 나가기 / 시스템 back: 생성 API 요청 취소 후 홈 이동
- 뒤로가기 아이콘 **없음**
- 중앙: 로딩 이미지용 **빈/플레이스홀더 컨테이너** + `콘텐츠 발행 중이에요` + 크레딧 안내 문구
- 하단 CTA **미표시**

## 6. 엣지 케이스 & 에러 처리

| 케이스 | 기대 동작 |
|--------|----------|
| 1·2단계 필수 선택 미완료 | `다음` 비활성. 탭해도 진행하지 않음 |
| 3단계 강조 내용 미입력(공백 포함) | `콘텐츠 생성하기` 비활성 |
| 키워드 공백 추가 시도 | 추가하지 않음. 입력 필드 유지 또는 클리어 |
| 동일 키워드 중복 추가 | 무시하거나 기존 유지 (구현 시 하나로 통일) |
| 네트워크 오류(생성 API) | 오류 안내 후 생성 중 화면에서 재시도 또는 3단계로 복귀 가능하도록 처리 |
| 생성 요청 중 나가기 | 홈으로 이동. 서버 요청 취소 정책은 TDD에서 결정 |
| 인증 만료 | 기존 전역 인증 정책(리프레시/로그인 유도)을 따름 |
| 생성 중 시스템 뒤로가기(Android) | 앱바 back이 없으므로, 시스템 back은 나가기와 동일하게 홈 이동 또는 ignore — TDD에서 확정 |

## 7. 성공 지표 (Success Metrics)

- 사용자가 필수 입력을 충족한 경우에만 다음/생성 CTA가 활성화된다.
- 키워드 추가·삭제 후 칩 목록이 UI·요청 payload와 일치한다.
- `콘텐츠 생성하기` 탭 시 수집된 enum·텍스트·키워드·사진 가이드 여부가 API 요청에 포함된다.
- 생성 중 화면에서 뒤로가기 없이 `나가기`만으로 홈 복귀가 가능하다.

## 8. 의존성 & 선행 조건

- **API**: 콘텐츠 생성 요청 엔드포인트 (채널·목적·톤·강조/금지 내용·키워드·사진 가이드 여부 등)
- **권한**: 로그인(인증)된 사용자
- **외부 서비스**: AI 콘텐츠 생성 백엔드
- **선행 기능**: 인증/세션, 홈 라우팅, 공통 버튼·앱바·디자인 토큰
- **디자인**: 1단계(업로드 채널) hi-fi Figma 확보 시 화면 목록·옵션을 최종 확정. 현재는 와이어프레임(블로그/스레드/당근/인스타그램) 기준
