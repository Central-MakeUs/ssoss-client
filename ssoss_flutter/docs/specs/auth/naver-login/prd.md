## 1. 배경 및 목적

**배경**

이메일/비밀번호 방식보다 마찰이 적은 소셜 간편 로그인이 진입 장벽을 낮추는 데 효과적이다.

**목적**

국내 사용자에게 익숙한 네이버 계정 기반 간편 로그인을 제공하여 가입/로그인 과정의 마찰을 최소화한다. 네이버 로그인을 우선 구현하되, 이후 카카오·애플 등 다른 소셜 로그인으로 확장 가능한 구조를 전제로 설계한다.

---

## 2. 사용자 스토리

- [ ] As a **신규 사용자**, I want to **네이버 계정으로 별도 회원가입 없이 로그인**, so that **빠르게 서비스를 시작할 수 있다**.
- [ ] As a **기존 사용자**, I want to **앱 재실행 시 다시 로그인하지 않고 자동으로 인증 상태가 유지**, so that **매번 로그인하는 번거로움 없이 서비스를 이용할 수 있다**.
- [ ] As a **로그인한 사용자**, I want to **로그아웃**, so that **내 계정을 안전하게 보호할 수 있다**.
- [ ] As a **로그인한 사용자**, I want to **회원 탈퇴(=소셜 연동 해제)**, so that **내 계정 연결과 인증 상태를 완전히 정리할 수 있다**.

## 3. 요구사항

### 3.1 기능 요구사항 (Functional Requirements)

| ID | 요구사항 | 우선순위 |
|----|---------|---------|
| FR-01 | 로그인 화면에 네이버 로그인 버튼을 노출하고, 탭 시 네이버 인증 흐름을 시작한다. | Must |
| FR-02 | 네이버 SDK로 획득한 토큰/인가코드를 자체 백엔드에 전달하고, 자체 JWT(access/refresh)를 발급받는다. | Must |
| FR-03 | 발급받은 access/refresh 토큰을 secure storage에 안전하게 저장한다. | Must |
| FR-04 | 앱 재실행 시 저장된 토큰으로 자동 로그인(세션 복원)을 수행한다. | Must |
| FR-05 | 사용자가 로그아웃하면 앱의 저장 토큰을 삭제하고 네이버 SDK 세션을 해제한 뒤 로그인 화면으로 이동한다. (연동 revoke는 하지 않음) | Must |
| FR-06 | 사용자가 네이버 인증을 취소하거나 실패한 경우, 로그인 화면을 유지하고 안내 메시지를 표시한다. | Must |
| FR-07 | 어떤 API 호출이든 access 토큰 만료가 감지되면 refresh 토큰으로 재발급 후 원요청을 재시도한다. (전역 인터셉터) | Should |
| FR-08 | 소셜 로그인 provider를 추상화하여 이후 카카오·애플 로그인을 추가할 수 있는 구조로 구성한다. | Should |
| FR-09 | 미가입 신규 사용자의 경우 백엔드 정책에 따라 자동 회원 생성 또는 온보딩으로 분기한다. | Could |
| FR-10 | 사용자가 탈퇴를 요청하면(확인 절차 포함) 앱 세션/토큰을 즉시 폐기하고 로그인 화면으로 이동한다. | Must |
| FR-11 | 백엔드 연동 시 탈퇴 API 호출에 성공하면 서버 계정 상태를 탈퇴로 전환하고, **서버에서 네이버 연동 revoke** 를 수행한다. | Should |

> **우선순위 기준** — Must: 반드시 / Should: 가급적 / Could: 여유 시

> **데모 단계 참고**
> - FR-10(데모): 로컬 세션/토큰 삭제만 수행 (클라이언트 SDK `logoutAndDeleteToken` 호출 없음)
> - FR-11: 서버 처리 — 네이버 연동 revoke는 서버에서 수행

### 3.2 비기능 요구사항 (Non-functional Requirements)

| 항목 | 요구사항 |
|------|---------|
| 성능 | 네이버 인증 완료 후 홈 진입까지 정상 네트워크 기준 3초 이내 |
| 보안 | access/refresh 토큰은 secure storage에 저장, 모든 인증 통신은 HTTPS. 토큰을 로그·평문 저장소에 노출하지 않는다. |
| 오프라인 | 오프라인 시 로그인 시도 불가 안내. 유효 세션이 있으면 캐시된 인증 상태로 진입 허용(정책에 따름) |
| 접근성 | 네이버 로그인 버튼에 스크린 리더용 라벨 제공, 최소 터치 영역 확보 |

## 4. 범위 (Scope)

### In Scope (이번 구현에 포함)

- 네이버 로그인 버튼 및 인증 흐름 연동
- 네이버 토큰 → 자체 백엔드 전달 → 자체 JWT(access/refresh) 발급
- 토큰 secure storage 저장 및 자동 로그인(세션 복원)
- 로그아웃 (로컬 토큰 삭제 + 네이버 SDK 세션 해제)
- 회원 탈퇴(데모 범위: 로컬 세션/토큰 삭제)
- 인증 취소/실패 처리
- 이후 소셜 로그인 확장을 고려한 provider 추상화 구조

### Out of Scope (이번 구현에서 제외)

- 카카오·구글 등 다른 소셜 로그인 (이유: 네이버 로그인 우선 검증 후 확장 예정. 단, 구조는 확장 가능하게 설계)
- 애플 로그인 (이유: 별도 스펙으로 분리 — [`apple-login/prd.md`](../apple-login/prd.md) 참조)
- **탈퇴 시 클라이언트 네이버 연동 revoke (`logoutAndDeleteToken`)** (이유: 서버에서 처리 예정)
- 이메일/비밀번호 로그인 및 자체 회원가입 폼 (이유: 소셜 로그인 우선 정책)
- 회원 탈퇴/계정 삭제 정책(데이터 보존·복구 정책)은 별도 Account 도메인 스펙으로 분리
- 프로필 조회/편집 (이유: 별도 기능 스펙으로 분리)

## 5. 화면 & UX 흐름

**화면 목록**

| 화면 이름 | 설명 | Figma 링크 |
|-----------|------|-----------|
| 로그인 화면 | 네이버 로그인 버튼을 제공하는 진입 화면 (`lib/features/auth/presentation/pages/login_page.dart`) | N/A |
| 네이버 인증 화면 | 네이버 SDK/웹뷰가 제공하는 인증 화면 (외부 제공) | N/A |
| 홈 화면 | 로그인 성공 후 이동하는 서비스 진입 화면 | N/A |

**주요 사용자 흐름**

```
[로그인 화면] → 네이버 로그인 버튼 탭 → [네이버 인증 화면] → 인증 승인
  → 네이버 토큰/인가코드 획득 → 자체 백엔드 전달 → 자체 JWT 발급
  → 토큰 secure storage 저장 → [홈 화면]

[앱 재실행] → 저장 토큰 확인
  → 유효 → 세션 복원 → [홈 화면]
  → 만료/없음 → [로그인 화면]

[홈/설정] → 로그아웃 → 로컬 토큰 삭제 + 네이버 SDK 세션 해제(연동 유지) → [로그인 화면]

[홈/설정] → 회원 탈퇴 탭 → 확인 다이얼로그(되돌릴 수 없음)
  → (데모) 로컬 세션/토큰 삭제 → [로그인 화면]
  → (백엔드 연동) 탈퇴 API 성공 시 서버 계정 탈퇴 + 서버에서 네이버 연동 revoke
```

## 6. 엣지 케이스 & 에러 처리

| 케이스 | 기대 동작 |
|--------|----------|
| 사용자 인증 취소 | 로그인 화면 유지, 별도 에러 없이 원상 복귀 |
| 네트워크 오류 | 토스트/스낵바로 오류 안내 후 재시도 가능 상태 유지 |
| 네이버 토큰 만료·무효 | 네이버 재인증 유도 |
| 백엔드 JWT 발급 실패 | 오류 안내 후 로그인 화면 유지, 재시도 제공 |
| access 토큰 만료 | 전역 인터셉터가 refresh API 호출 후 원요청 재시도, 갱신 실패 시 로그아웃 처리 후 로그인 화면 이동 |
| refresh 토큰 만료·무효 | 세션 종료, 로그인 화면으로 이동 |
| 미가입 신규 사용자 | 백엔드 정책에 따라 자동 가입 또는 온보딩 분기 |
| 탈퇴 API 실패(백엔드) | 오류 안내 후 현재 화면 유지, 재시도 또는 고객센터 경로 제공 |
| 탈퇴 처리 중 앱 종료 | 앱 재실행 시 세션 상태 재검증. 탈퇴 완료 시 로그인 화면 강제 이동 |

## 7. 성공 지표 (Success Metrics)

- 네이버 로그인 시도 대비 성공률이 목표치(예: 95%) 이상이다.
- 유효 세션 보유 사용자의 자동 로그인(세션 복원) 성공률이 99% 이상이다.
- 인증 취소/실패 시 앱이 크래시 없이 로그인 화면으로 안전하게 복귀한다.
- 탈퇴 요청 성공 시 로컬 세션 잔존율(토큰 미삭제) 0%를 유지한다.

## 8. 의존성 & 선행 조건

- **API**: 백엔드의 소셜 로그인 인증 엔드포인트(네이버 토큰 수신 → 자체 JWT 발급), 토큰 갱신·탈퇴 엔드포인트. 네이버 연동 revoke 는 **서버**에서 처리
- **권한**: 네트워크 통신 권한 (기본), 네이버 앱 연동 시 필요한 URL scheme/딥링크 설정
- **외부 서비스**: 네이버 개발자센터 앱 등록 및 Client ID / Client Secret 발급, 네이버 로그인 SDK
- **선행 기능**: 환경 변수를 통한 Client ID/Secret 및 API Base URL 관리, secure storage 패키지 도입, 인증 상태에 따른 라우팅 가드(`lib/router/app_router.dart`)
